# Sportivo Roadmap

Sportivo has gone through a full cleanup and portfolio-polish process. The current version is ready for final local testing, GitHub cleanup, and static deployment.

## Completed improvements

- Created `index.html` as the main site entry point.
- Prepared the project for static deployment.
- Replaced placeholder product graphics with real sports product photography.
- Expanded the catalog with richer product data.
- Improved search, sport filtering, category filtering, price filtering, sorting, quick filters, and active filter chips.
- Redesigned the homepage around a real storefront experience.
- Added a custom Sportivo logo and brand assets.
- Removed storefront-facing technical wording from the UI.
- Improved product details with related products, recently viewed products, buyer-confidence details, wishlist, add-to-cart, and buy-now actions.
- Improved checkout with customer details, shipping details, payment selection, validation, and order confirmation.
- Improved orders and tracking with status summaries, package timelines, buy-again actions, and cleaner post-purchase UI.
- Added page-aware headers so each page has the correct navigation level.
- Replaced the old review modal with a cleaner buyer-confidence section.
- Unified product card styling across the storefront.
- Improved responsive behavior and accessibility.
- Added final documentation for deployment, QA, and CV/portfolio usage.

- Add a lightweight build tool such as Vite for asset optimization.
- Add automated linting and formatting.
- Add unit tests for cart, wishlist, filter, and order utilities.
- Add a real backend API for products, authentication, orders, and reviews.
- Add admin product management.
- Add real payment integration in a production-safe environment.
- Add image optimization pipeline and responsive image variants.
